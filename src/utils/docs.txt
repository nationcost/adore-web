================================================================================
                           2. EMBED SYSTEM
================================================================================

The bot supports two types of rich messages:
  ‚Ä¢ Embeds     - Traditional Discord embeds with title, description, fields, etc.
  ‚Ä¢ Containers - Discord Components V2 with text, sections, galleries, buttons

Both can be used in: createembed, autoresponder, welcome, leave, boost messages

--------------------------------------------------------------------------------
                          BASIC EMBED SYNTAX
--------------------------------------------------------------------------------

Format: {embed}$v{property: value}$v{property: value}...

Each property is separated by $v and wrapped in curly braces {}.

EMBED PROPERTIES:
-----------------

color:
  Sets the embed color (hex code)
  Syntax: $v{color: #hex}
  Example: $v{color: #ff5733}

author:
  Sets the author field with optional icon and URL
  Syntax: $v{author: name && icon_url && url}
  Example: $v{author: John Doe && https://example.com/avatar.png}
  Example: $v{author: John Doe && https://example.com/avatar.png && https://example.com}

title:
  Sets the embed title
  Syntax: $v{title: text}
  Example: $v{title: Welcome to the Server!}

description:
  Sets the embed description (main body text)
  Syntax: $v{description: text}
  Example: $v{description: This is the main content of the embed}

thumbnail:
  Sets a small image in the top-right corner
  Syntax: $v{thumbnail: url}
  Example: $v{thumbnail: https://example.com/image.png}

image:
  Sets a large image at the bottom of the embed
  Syntax: $v{image: url}
  Example: $v{image: https://example.com/banner.png}

footer:
  Sets the footer text with optional icon
  Syntax: $v{footer: text && icon_url}
  Example: $v{footer: Thanks for joining!}
  Example: $v{footer: Thanks for joining! && https://example.com/icon.png}

field:
  Adds a field with name, value, and optional inline setting
  Syntax: $v{field: name && value && inline}
  Example: $v{field: Rules && Follow the rules! && false}
  Example: $v{field: Members && 100 && true}
  Note: inline can be "true" or "false" (default: false)

timestamp:
  Adds the current timestamp to the embed
  Syntax: $v{timestamp}
  Example: $v{timestamp}

content: / message:
  Sets text content outside the embed (above it)
  Syntax: $v{content: text} or $v{message: text}
  Example: $v{content: Hey everyone!}

FULL EMBED EXAMPLE:
-------------------
{embed}$v{color: #5865F2}$v{author: Server Bot && {guild.icon}}$v{title: Welcome!}$v{description: Welcome to **{guild.name}**, {user.mention}!}$v{thumbnail: {user.avatar}}$v{field: Members && {guild.count} && true}$v{field: Rules && Check #rules && true}$v{image: {guild.banner}}$v{footer: Enjoy your stay! && {guild.icon}}$v{timestamp}

--------------------------------------------------------------------------------
                          CONTAINER SYNTAX (Components V2)
--------------------------------------------------------------------------------

Format: {container}$v{property: value}$v{property: value}...

Containers are Discord's newer component-based messages with more flexibility.

CONTAINER PROPERTIES:
---------------------

color:
  Sets the accent color of the container
  Syntax: $v{color: #hex}
  Example: $v{color: #ff5733}

text:
  Adds a text display element
  Syntax: $v{text: content}
  Example: $v{text: Welcome to the server!}

separator:
  Adds a visual separator line
  Syntax: $v{separator} or $v{separator:large}
  Example: $v{separator}
  Example: $v{separator:large}

section:
  Adds a section with text and optional thumbnail
  Syntax: $v{section: text && thumbnail_url}
  Example: $v{section: Check out our rules!}
  Example: $v{section: User Profile && {user.avatar}}

image:
  Adds a single image to the container
  Syntax: $v{image: url}
  Example: $v{image: https://example.com/image.png}

gallery:
  Adds multiple images in a gallery
  Syntax: $v{gallery: url1 && url2 && url3}
  Example: $v{gallery: https://example.com/1.png && https://example.com/2.png}

content: / message:
  Sets text content above the container
  Syntax: $v{content: text} or $v{message: text}
  Example: $v{content: Hey {user.mention}!}

FULL CONTAINER EXAMPLE:
-----------------------
{container}$v{color: #5865F2}$v{content: Welcome!}$v{text: # Welcome to {guild.name}!}$v{separator}$v{section: {user.name} just joined! && {user.avatar}}$v{separator:large}$v{image: {guild.banner}}$v{button: https://discord.gg/example && Join Discord}

--------------------------------------------------------------------------------
                              BUTTONS
--------------------------------------------------------------------------------

Buttons can be added to both embeds and containers.

Syntax: $v{button: style_or_url && label && emoji}

BUTTON STYLES:

--------------------------------------------------------------------------------
                              FLAGS
--------------------------------------------------------------------------------

Flags modify how messages are sent. Add them at the END of your message.

AVAILABLE FLAGS:
----------------

--reply
  Makes the bot reply to the triggering message (autoresponder only)
  Example: Hello {user.mention}! --reply

--not_strict
  Triggers on partial matches instead of exact matches (autoresponder only)
  Without: "hello" only triggers on exactly "hello"
  With: "hello" triggers on "hello there", "say hello", etc.
  Example: Hi there! --not_strict

--delete N
  Deletes the bot's response after N seconds (1-10) (autoresponder only)
  Example: This message will disappear! --delete 5

--mention
  Allows @everyone, @here, and role mentions to actually ping
  Without this flag, mentions are displayed but don't ping
  Example: {embed}$v{description: @everyone check this out!} --mention

COMBINING FLAGS:
----------------
You can combine multiple flags:
  Hello {user.mention}! --reply --not_strict --delete 5
